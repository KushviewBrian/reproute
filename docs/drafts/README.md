# Docs Drafts

Use this folder for planning and audit documents that are useful context but are not canonical source-of-truth specs.

Keep in `docs/` root only documents that are:
- referenced by `docs/roadmap.md`, README, gate templates, or runbooks, and
- expected to remain stable project references.

Move drafts here when they are:
- exploratory plans,
- temporary review/audit notes,
- superseded versions of canonical docs.
